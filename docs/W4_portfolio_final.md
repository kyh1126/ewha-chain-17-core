# W4 개별 포트폴리오 완성 가이드

**기간**: 3월 4주  
**목표**: Core 산출물 마무리 + Demo Video 초안 제작 + 리포트 작성 + Extension 트랙 선택 확정

## 📋 W4 체크리스트

- [ ] 대시보드 최종 완성
- [ ] Demo Video 제작 (2~3분)
- [ ] 요약 리포트 작성 (1~2p)
- [ ] 코드 리뷰(PR) 완료 또는 요청
- [ ] 최종 발표 리허설 참여 (각자 5분)
- [ ] Extension 트랙 선택 확정

## 1. 팀장 역할 (참고)

- **코드 리뷰**: PR 기반으로 쿼리·스크립트 리뷰
- **Demo Video 가이드**: 본 문서의 Demo Video 섹션 및 [리포트 템플릿](#4-요약-리포트-작성-12p) 활용
- **리포트 템플릿**: 아래 "요약 리포트 작성" 참고

## 2. 대시보드 최종 완성 (개별 작업)

### 2.1 필수 구성

- **표준 지표 3~4개** 반영 (거래량, 활성 주소, 수수료/가스, 실패 트랜잭션 등)
- **개별 지표 1개 이상** 포함
- Looker Studio(또는 웹) 대시보드 1개, 공유 가능한 링크

### 2.2 정리 사항

- 차트 제목·축 레이블·기간 표시 명확히
- `members/[본인이름]/dashboard/` 에 스크린샷 또는 링크 정리
- `members/[본인이름]/README.md` 에 대시보드 링크 추가

**✅ 확인**: 대시보드 링크가 팀원/팀장이 열어볼 수 있는지 확인

## 3. Demo Video 제작 (2~3분)

### 3.1 권장 구성 및 시간

| 구간 | 내용 | 권장 시간 |
|------|------|-----------|
| 1. 문제 정의 | 어떤 질문/지표를 다루는지 | 약 30초 |
| 2. 대시보드 시연 | 차트 설명, 필터·기간 변경 시연 | 약 1분 |
| 3. Gemini 요약 | 요약 생성 과정 또는 결과 읽기 | 약 30초 |
| 4. 결론/인사이트 | 핵심 결론 3줄, 기관 관점 코멘트 | 약 30초 |

총 **2~3분** 이내로 구성하세요.

### 3.2 제작 방법

- **녹화 도구**: OBS, Loom, Zoom 녹화, 휴대폰 등 자유
- **음성**: 설명을 음성으로 넣는 것을 권장 (자막만 있어도 가능)
- **화면**: BigQuery 콘솔, Looker Studio, 터미널(Python 실행), 리포트 한 장 등

### 3.3 제출

- **파일**: `deliverables/demo_videos/[본인이름]_demo.mp4`
- **또는** YouTube 등 업로드 후 링크를 `deliverables/demo_videos/[본인이름]_link.txt` 에 저장

자세한 제출 방법: [deliverables/README.md](../deliverables/README.md)

**✅ 확인**: 2~3분 분량, 문제 정의 → 대시보드 → Gemini → 결론 순서 포함

## 4. 요약 리포트 작성 (1~2p)

### 4.1 목적

- **기관 관점**에서 Solana(및 비교 체인)의 처리량·패턴을 데이터로 설명하는 한 페이지 요약
- Demo Video와 함께 포트폴리오를 설명하는 자료

### 4.2 리포트 템플릿 (1~2p)

아래 구조를 참고해 1~2페이지 분량으로 작성하세요.

```markdown
# [본인이름] Core 요약 리포트

## 1. 분석 목표 (2~3문장)
- 어떤 질문에 답하려 했는지
- 어떤 지표를 사용했는지

## 2. 데이터 및 방법
- 사용한 데이터셋 (Ethereum / Solana, 기간)
- 핵심 쿼리/지표 3~4개 요약

## 3. 주요 결과 (3~5문장)
- 대시보드에서 도출한 핵심 인사이트
- 전주 대비 변화, 이상 징후 등 (해당 시)

## 4. Gemini 요약 요약
- 생성된 주간 요약 또는 비교 결론 3줄을 요약하여 인용

## 5. 결론 및 한계 (2~3문장)
- 기관 관점에서의 결론
- 데이터/기간 한계 등 (선택)
```

### 4.3 제출

- **파일**: `deliverables/reports/[본인이름]_report.pdf` 또는 `[본인이름]_report.md`
- PDF 권장 (공유·평가용)

**✅ 확인**: 1~2p, 기관 관점 템플릿 구조 반영

## 5. 코드 리뷰 (PR 기반)

### 5.1 PR에 포함할 내용

- `members/[본인이름]/` 변경 사항 (쿼리, 스크립트, README)
- (선택) `deliverables/` 에 추가한 링크·파일

### 5.2 PR 제목/설명 예시

- 제목: `[본인이름] W4 Core 산출물 제출`
- 설명: 분석 목표, 대시보드 링크, Demo Video/리포트 위치 간단히 기재

팀장/동료의 코드 리뷰를 받고, 피드백 반영 후 머지하세요.

**✅ 확인**: PR 생성 완료, 리뷰 요청 또는 완료

## 6. 공유 세션: 최종 발표 리허설 (각자 5분)

3. Gemini 요약 결과 요약 (30초)
4. 결론 및 Extension 트랙 선택 (1분)
5. Q&A

### 6.2 준비 사항

- 대시보드 링크 열어 두기
- Demo Video 또는 리포트 1~2p 공유 화면 준비
- Extension 트랙(A: dApp·풀스택 / B: Node Engine) 중 선택한 트랙과 이유 간단히 정리

**✅ 확인**: 리허설 참여, 피드백 수용

## 9. 자주 묻는 질문 (W4)

**Q: 쿼리가 10개가 안 돼요.**  
A: 템플릿 3~4개 + 나만의 지표 1개 + 기타 분석용 쿼리를 합쳐 10개 이상을 목표로 하되, **최소 필수는 템플릿 기반 3개 + 대시보드 + Demo Video + 리포트**입니다.

**Q: Demo Video에 음성이 꼭 필요할까요?**  
A: 음성을 권장하지만, 자막만 있어도 됩니다. 내용(문제 정의 → 대시보드 → Gemini → 결론)이 들어가는 것이 중요합니다.

**Q: 리포트를 영어로 써도 되나요?**  
A: 프로그램 안내에 따르면 기관 관점 요약이므로 한국어를 기본으로 하되, 팀장/운영진에 확인하세요.
